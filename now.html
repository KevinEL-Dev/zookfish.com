More updates regarding rust cli/daemon project

So in order for my program to monitor how much resources hytale is using I need to know every hytale process that is running and take the sum cpu%. This seems like a lot of work but for now I found a command that should get the job done.

top -b -d 1 -n 2 | awk '$1 == "PID" {block_num++; next} block_num == 2 {sum += $9;} END {print sum}'

This command basically runs top, where top then prints a first snapshot of all processes. Top then waits a second and then prints another snapshot of the processes. From there we pipe that output and use awk. I heard of awk but I forgot what it does. For now I know that it does take the cpu% of each process that has the PID and its children.

This will be the plan to try and get the cpu% of all hytale processes and its children. So the first step is that I need to get the PID of the parent hytale process. We just use the command:

ps aux | grep java -XX:AOTCache=HytaleServer.aot

This will include the grep process as well so we pipe this into the command:

head -n 1

This should give us the parent process that hytale runs.
